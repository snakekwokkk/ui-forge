# Selection and extension interaction

Use a staged interaction so users compare direction before paying the cost of full extension.

## Gate 1: primary home option

After presenting A, B, and C home previews, stop and request exactly one choice:

- `A — Product Clear`
- `B — Brand Balanced`
- `C — Campaign Bold`
- `Adjust the options`

Prefer a structured single-select control when available. Do not continue automatically. If the runtime cannot render structured choices, show the same four choices as a short text list.

## Gate 2: confirmation and Figma target

After selection, show `Confirm <option>` and request a Figma file or node link if none was supplied. Also allow `Back to options`. Continue only after the direction and Figma destination are known.

## Primary extension

Extend the confirmed direction across all remaining wireframes, build the reusable components and 12–18 item candidate asset library, write the result to Figma, and complete visual QA.

## Gate 3: additional directions

After QA, stop and offer:

- `Finish with <selected option>`
- `Revise <selected option>`
- `Extend <remaining option 1>`
- `Extend <remaining option 2>`
- `Extend both remaining options`

Treat later extensions as new confirmed work. Preserve the already approved direction and build additional directions non-destructively in separate Sections.
